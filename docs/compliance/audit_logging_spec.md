# Audit & Compliance Logging Specification

The NexBank Agentic AI must adhere to stringent logging requirements under the RBI Digital Lending Guidelines, DPDP Act, and PCI DSS. This specification defines the structured schemas for conversation and turn-level logging, along with data retention and encryption standards.

## 1. Conversation-Level Log Schema

Each completed conversation produces a single structured log record.

| Field | Type | Description | Retention |
|---|---|---|---|
| `conversation_id` | UUID | Unique identifier for the conversation session | 7 years |
| `customer_id` | UUID (hashed) | Pseudonymised customer identifier | 7 years |
| `channel` | Enum | Communication channel (`chat`, `voice`, `whatsapp`, `app`) | 7 years |
| `authentication_level` | Enum | Highest authentication achieved (`unauthenticated`, `otp`, `biometric`) | 7 years |
| `primary_intent` | String | Final classified intent for the conversation (e.g., `ACC-001`) | 7 years |
| `resolution_status` | Enum | `resolved_ai`, `resolved_human`, `unresolved`, `escalated` | 7 years |
| `guardrails_activated` | Array[String] | List of guardrails that fired during conversation | 7 years |
| `csat_score` | Float | Customer satisfaction rating (1.0 - 5.0) if provided | 3 years |
| `model_version` | String | AI model version that handled the conversation | 7 years |
| `turn_count` | Integer | Total number of conversation turns | 7 years |
| `pii_detected` | Boolean | Whether PII was detected in customer messages | 7 years |

## 2. Turn-Level Log Schema

In addition to the conversation summary, every single dialogue turn is logged individually.

| Field | Type | Description |
|---|---|---|
| `turn_id` | UUID | Unique identifier for the specific turn |
| `speaker` | Enum | `customer`, `agent`, or `system` |
| `raw_message` | String (Encrypted) | The raw text, with all PII scrubbed/redacted |
| `intent_classification` | Object | `{ label, confidence, alternatives }` |
| `entities_extracted` | Array[Object] | `[{ type, value, confidence, validation_status }]` |
| `sentiment_score` | Float | Sentiment value from -1.0 to 1.0 |
| `knowledge_items_retrieved`| Array[String] | Chunk IDs retrieved by the RAG pipeline |
| `retrieval_confidence` | Float | RRF confidence score of retrieved knowledge |
| `agent_response` | String | The exact text generated by the agent |
| `response_generation_method`| Enum | `template`, `llm`, or `hybrid` |
| `guardrails_checked` | Array[String] | Guardrails evaluated during this turn |
| `guardrails_triggered` | Array[String] | Guardrails that actively blocked or altered the response |
| `response_latency_ms` | Integer | Time in milliseconds to generate the response |
| `action_taken` | Enum | `respond`, `clarify`, `confirm`, `escalate`, `authenticate` |

## 3. Compliance Requirements

### Data Encryption
- **At Rest**: All log data encrypted at rest using AES-256 encryption.
- **In Transit**: All log data encrypted in transit using TLS 1.3.
- **PII Handling**: Any residual PII fields are additionally encrypted with a separate key (double encryption).
- **Key Rotation**: Encryption keys rotated quarterly through a Hardware Security Module (HSM).

### Access Control
- Log access follows the **principle of least privilege**.
- Customer-facing staff access only anonymized, aggregated metrics.
- Supervisors access full logs *only* for conversations actively in their review queue.
- All log access itself is logged (a meta-audit trail).

### Data Retention & Deletion (DPDP Act)
- Retention periods (7 years for financial logs) are enforced automatically by the storage layer.
- Customer right-to-deletion requests are processed within 30 days.
- Deletion includes all copies, backups, and derived data (including model training data).
- Deletion certificates are generated and stored for regulatory proof.

## 4. Regulatory Reporting

The system automatically generates the following reports:
1. **Monthly RBI Report**: AI interaction summary detailing volume, escalation rates, and complaint categories.
2. **Quarterly Ombudsman Analysis**: Customer complaint analysis matching Banking Ombudsman requirements.
3. **Annual AI System Audit**: Comprehensive report covering accuracy, fairness, and safety metrics.
4. **Ad-Hoc Queries**: System supports generating specific regulatory query responses within 48 hours of receipt.
